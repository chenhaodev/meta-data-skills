## Dataset Overview

**Hospital@Home** is a real clinical dataset from the Brigham and Women's / Faulkner Hospital
(BWH/BWFH) Hospital@Home programme, published by **Levine et al. (Ann Intern Med 2020)**.

- **574 patients** enrolled in the Hospital@Home RCT (2016–2017)
- **140 columns** covering 4 domains: admission metadata, clinical conditions & medications,
  demographic covariates, and health-related quality of life (HRQoL)
- Modality: **structured tabular** (Excel, no free text), English labels
- No ground-truth outcome prediction label — intended for patient **simulation** and
  **clinical profile generation**

Key column groups:

| Group | Columns |
|-------|---------|
| Admission | `Biofourmis_ID`, `ED`, `HH_Admission_#`, `Admitting_from`, `Disposition_ED` |
| Clinical conditions | `Condition_1–5`, `Condition_Chronic_Count`, `Comorbidities_Count` |
| Medications | `Current_Meds1–26`, `Medications_outpatient` |
| Demographics | `Age`, `Gender`, `Race_ethnicity`, `Language`, `Insurance`, `Education` |
| Functional status | `ADL_A_tot`, `IADL_A_tot`, `Delta_ADL_MINUS30toA` |
| HRQoL | `EQ_A_VAS` (EuroQoL VAS 0–100), `EQ_A_1–5` (EQ-5D dimensions) |
| Frailty/cognition | `PRISMA_tot` (frailty screen), `AD8_tot` (dementia), `BRIEF_tot` |
| Mood | `PHQ_A_1–2`, `PHQ_A_tot` (PHQ-2 depression screen) |
| Social support | `Support_A_tot`, `Lives_alone`, `Partner_status` |

Paper: Levine DM, Ouchi K, Blanchfield B, et al. *Hospital-Level Care at Home for Acutely Ill
Adults: A Randomized Controlled Trial.* Ann Intern Med. 2020;172(2):77-85.
DOI: 10.7326/M19-0600

## Clinical Domain & AI Relevance

**Domain:** Acute internal medicine, hospital medicine, nursing assessment, care coordination.

**Conditions represented:** Rheumatoid arthritis, asthma, HTN, pneumonia, CHF, COPD, cellulitis,
PE, UTI, and other acutely decompensated chronic conditions typical of hospital-level care.

**Why this matters for LLM evaluation:**
- **Patient simulation:** The richest use — feed a patient profile to an LLM acting as a virtual
  patient; evaluate nursing/physician LLM responses against clinical best practice.
- **Clinical prediction proxy:** Predict fall risk (`Morse_Fall_Risk_Score`), frailty (`PRISMA_tot`),
  or ADL decline (`Delta_ADL_MINUS30toA`) from demographics + conditions + meds.
- **Real data, not synthetic:** Unlike most LLM medical benchmarks, this is de-identified real EHR
  and patient-reported data from an RCT — high ecological validity.

No published LLM benchmark scores exist for this dataset; it is used as a **simulation substrate**,
not a scored leaderboard benchmark.

There are no official task subtasks. Common analytical subsets:

- **Fall-risk patients:** `Morse_Fall_Risk_Score >= 45`
- **Frail patients:** `PRISMA_tot >= 3`
- **Cognitively impaired:** `AD8_tot >= 2`
- **High medication burden:** `Medications_outpatient >= 10`
- **ADL decliners:** `Delta_ADL_MINUS30toA < 0`

## Recommended Use Cases

- **Patient simulation / benchmarking:** Use `format_item()` to build a natural-language patient
  profile. Prompt an LLM to act as the patient; evaluate clinical reasoning of a physician/nurse
  LLM responding to it. Score responses with a rubric or expert review.
- **Clinical prediction tasks:** Treat `Morse_Fall_Risk_Score`, `PRISMA_tot`, or `EQ_A_VAS` as
  prediction targets; use conditions + demographics + meds as features for LLM zero-shot or
  fine-tuned classification.
- **Few-shot prompt selection:** Sample patients stratified by `Condition_Chronic_Count` to cover
  complexity ranges (simple = 1 condition, complex = 5+).
- **Agentic evaluation:** Build an agentic nursing assistant that queries patient fields via tool
  calls; evaluate whether the agent correctly prioritises high-risk patients (fall risk, frailty).

## Pitfalls & Gotchas

1. **Header at row 8:** The Excel file has 8 metadata rows before the actual column names.
   Always load with `pd.read_excel(..., header=8)` — default `header=0` returns garbage column names.
2. **No outcome column:** There is no readmission, mortality, or LOS outcome variable in this
   extract. For prediction tasks you must define your own label from existing fields.
3. **High missingness in meds:** `Current_Meds6` and beyond have progressively fewer non-null
   values. Gather medications with a list comprehension over `Current_Meds1–26`, dropping NaN.
4. **Research-only data:** Do not share, publish, or upload this dataset anywhere. It contains
   de-identified real patient data from an RCT with a restricted data use agreement.
5. **No temporal data:** This is a cross-sectional snapshot at admission — no time-series vitals
   or labs are included despite the Biofourmis wearable connection in the original trial.
