# MedBench 2026 Dataset Expert

> This skill was generated by `/meddata:build medbench-2026`.

## Dataset Overview

**MedBench 2026** (MedBench V4) is a comprehensive **Chinese-language agentic medical benchmark**
created by the MedBench consortium (2026). It evaluates LLMs across 14 clinically grounded subtasks
covering the full spectrum of agentic medical AI capabilities.

- **420 test items** total (14 subtasks × 30 items each)
- **Language:** Chinese (Simplified) — questions, answers, and READMEs are all in Chinese
- **Modality:** text only
- **Format:** JSONL, one file per subtask in `MedBench_Agent/TEST/`
- **Evaluation:** LLM-as-judge for 12 subtasks; Accuracy for MedIntentID

Reference: https://www.emergentmind.com/topics/medbench-v4

## Clinical Domain & AI Relevance

**Why it matters:** MedBench 2026 is the most comprehensive agentic medical benchmark available
in Chinese, covering tasks that go far beyond MCQ: API calling, database operations, multi-system
collaboration, long-context reasoning, and safety/ethics. It directly mirrors real clinical workflow
challenges that deployed medical AI must handle.

**Subtask taxonomy:**

| Category | Subtasks | Metric |
|----------|----------|--------|
| Reasoning & CoT | MedCOT, MedReflect, MedDecomp | LLM-judge |
| Agentic tool use | MedCallAPI, MedRetAPI, MedDBOps | LLM-judge |
| Planning | MedPathPlan, MedCollab | LLM-judge |
| Long context | MedLongConv, MedLongQA | LLM-judge |
| Role adaptation | MedRoleAdapt | LLM-judge |

Published SOTA (Claude Opus 4.6, Feb 2026 internal submission): ~95% average across subtasks.

## Recommended Use Cases

- **Benchmarking LLMs:** Run each subtask with `load_dataset(subtask="MedCOT")`, collect
  model outputs, then score via LLM-as-judge (GPT-4o or Claude Sonnet). MedIntentID is the
  only subtask with automatic accuracy scoring.
- **Fine-tuning data selection:** Use subtask READMEs to understand each task's subcategory
  taxonomy; synthesise training data matching the same clinical scenario types.
- **Few-shot prompt selection:** Load a specific subtask and use `sample()` to get 3–5 examples
  to include in system prompts for zero-shot → few-shot comparison.
- **Agentic evaluation:** MedCallAPI, MedDBOps, MedRetAPI are ideal for testing tool-use and
  function-calling capabilities of agentic systems in Chinese medical contexts.

## Pitfalls & Gotchas

1. **Chinese-only content:** All 420 questions and ground-truth answers are in Simplified Chinese.
   English-only models will score near zero. Ensure your model has strong Chinese medical capability.
2. **Empty test answers:** The `answer` field in all JSONL test files is an empty string. There is
   no reference answer provided — scoring requires LLM-as-judge against the question rubric, or
   submission to the official MedBench leaderboard.
3. **MedDefend ≠ MedEthics:** The actual TEST file is `MedDefend.jsonl`, not `MedEthics.jsonl`
   (which appears in some documentation). Only 14 READMEs exist; MedDefend has no README.
4. **Subtask name vs filename:** Directory is `data/MedBench-2026/` (capital B, hyphen) not
   `data/medbench-2026/`. The `local_dir` in TARGET.md is relative to the data directory.
5. **No official scoring script:** The submission format (see `SUBMIT/` folder) is a JSONL with
   `question`/`answer` fields, but the judge prompt and scoring rubric are not open-sourced.
   Use the leaderboard API for official scores.
