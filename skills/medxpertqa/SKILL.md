## Dataset Overview

**MedXpertQA** is an expert-level medical MCQ benchmark from **TsinghuaC3I** (Tsinghua
University, 2025). It is specifically designed to challenge frontier models where existing
benchmarks (MedQA, MMLU) have been saturated.

- **4,460 questions** total across two subsets:
  - **Text** (2,460 questions): text-only, 10-option MCQ (A–J)
  - **MM** (2,010 questions): multimodal (text + images), 5-option MCQ (A–E)
- **17 medical specialties**, **11 body systems**, English language
- **Fields:** `id`, `question`, `options` (dict), `label` (answer letter), `images` (MM only),
  `medical_task`, `body_system`, `question_type`
- Sourced from specialty board exam questions with multiple rounds of expert review;
  data synthesis applied to reduce contamination risk

Paper: *MedXpertQA: Benchmarking Expert-Level Medical Reasoning and Understanding*
arXiv:2501.18362 (2025) — Zuo et al., TsinghuaC3I

## Clinical Domain & AI Relevance

**Medical task categories:**
- `Diagnosis` — identifying conditions from clinical presentation
- `Treatment` — selecting appropriate interventions
- `Basic Medicine` — foundational science (pharmacology, physiology, pathology)

**Question type categories:**
- `Reasoning` — multi-step clinical inference
- `Understanding` — knowledge recall and concept application

**Body systems covered:** Cardiovascular, Digestive, Nervous, Lymphatic, Reproductive,
Musculoskeletal, Respiratory, Endocrine, Urinary, Integumentary, and more.

**Why it matters:** MedXpertQA is one of the few benchmarks where frontier models score
well below 60%, making it a strong differentiator for today's leading models.

## Recommended Use Cases

- **Benchmarking LLMs (text):** Load `subset="Text"`, run `run_benchmark()` with your model
  callable returning a single letter (A–J). Compare against the SOTA table above. Random
  baseline is 10% — anything below 15% indicates near-random performance.
- **Benchmarking VLMs (multimodal):** Load `subset="MM"`; pass both question text and
  `item["metadata"]["images"]` filenames to your vision model. Random baseline is 20%.
- **Fine-tuning data selection:** Filter `question_type == "Reasoning"` for chain-of-thought
  fine-tuning data; filter `medical_task == "Diagnosis"` for diagnostic reasoning.
- **Few-shot prompt selection:** Sample from `dev` split (5 items per subset) for in-context
  examples, or stratify by `body_system` to match your target domain.
- **Agentic evaluation:** Use `Reasoning`-type questions to test multi-step thinking; compare
  performance with and without chain-of-thought prompting.

## Pitfalls & Gotchas

1. **Two incompatible option sets:** Text subset uses A–J (10 options); MM uses A–E (5 options).
   Always check `subset` before interpreting single-letter labels — a 'F' answer is valid in
   Text but invalid in MM.
2. **Images not bundled with HuggingFace load:** The MM `images` field contains filenames.
   Actual image bytes may need to be fetched separately via the HuggingFace dataset API
   (`ds[i]["images"]` may return PIL images if loaded with `trust_remote_code=True`).
3. **License ambiguity:** HuggingFace card says MIT; the arXiv paper is CC BY-NC 4.0.
   Treat as research-only / non-commercial to be safe.
4. **Contamination mitigation is partial:** Authors applied data synthesis but acknowledge
   residual leakage risk. Results on models trained after Jan 2025 should be interpreted
   cautiously.
5. **Dev split is tiny:** Only 5 items per subset — too small for few-shot selection.
   Use `sample()` from the test split instead for prompt engineering experiments.
